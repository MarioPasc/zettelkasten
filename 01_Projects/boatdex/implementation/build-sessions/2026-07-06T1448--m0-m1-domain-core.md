## Spec findings to raise (do NOT treat as code-truth)

These are inaccuracies found in the spec during implementation. They must be raised against the spec notes — the code pins the mathematically correct values, not the spec figures.

1. **`rarity-surprisal.md` golden table — bits column is slightly off.** The p̃ column is correct (0.0335884 = 79/2352; 0.0131803 = 31/2352) but −log₂(p̃) is ≈4.89586 and ≈6.24547 respectively, whereas the table prints 4.8957 and 6.2446. Tests pin the exact −log₂(p̃) values.
2. **`geodesy-identify.md` reciprocal-bearing claim is only exact on meridians / equator.** The claim "reciprocal bearing ≈ 180° for any a≠b (atol 1e-6)" holds exactly only on a shared meridian or the equator; in general the two initial bearings differ from 180° by ≈ Δlon · sin(lat) (meridian convergence). Tests pin the exact meridian/equator cases and verify the property over small separations (the camera-identify regime).
